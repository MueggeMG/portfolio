01 — Motivation

Warum dieses Projekt?

In Bewerbungen zählt der erste Eindruck. Ein Link zu einer eigenen Website sagt mehr als jeder Lebenslauf — er zeigt dass man nicht nur über Entwicklung redet, sondern sie auch wirklich macht.

Gleichzeitig wollte ich die Grundlagen wirklich verstehen. HTML, CSS und JavaScript ohne Framework, ohne Hilfsmittel — jede Zeile bewusst geschrieben und verstanden. Kein Copy-Paste, kein blindes Nachbauen.

Das Ergebnis ist eine Seite die nicht nur als Portfolio dient, sondern auch als Fundament für alle weiteren Projekte — gemeinsame CSS Variablen, ein einheitliches Design-System, wiederverwendbare Komponenten.

"Ich entwickle digitalen Fortschritt — präzise, modern und mit einem Blick für Details."

Als Entwickler in Ausbildung war es mir wichtig früh ein Projekt zu haben das ich zeigen kann — etwas das live ist, das funktioniert, und das meinen Stil widerspiegelt.

02 — Tech Stack

Was ich genutzt habe
und warum.

Struktur
HTML

Semantisches Markup. Jedes Element hat einen Zweck — keine überflüssigen Divs, keine generischen Wrapper. Sauber, lesbar, zugänglich.

Design
CSS

Kein Framework, kein Tailwind. CSS Variables für ein konsistentes Design-System, Grid und Flexbox für das Layout, eigene Animationen mit IntersectionObserver.

Interaktion
JavaScript

Vanilla ES6+. Custom Cursor mit LERP-Interpolation, Scroll-Animationen, Nav-Effekte — alles ohne externe Dependencies.

Design System
CSS Variables

Einmal definiert, überall genutzt. Farben, Abstände und Schriften in einer zentralen base.css — alle Projektseiten teilen dasselbe System.

Hosting
GitHub Pages

Kostenlos, schnell, versioniert. Jeder Commit ist automatisch live — kein manuelles Hochladen, keine Server-Konfiguration.

Typografie
Google Fonts

Cormorant Garamond + DM Mono. Serif für Überschriften, Monospace für Fließtext — eine Kombination die gleichzeitig elegant und technisch wirkt.

03 — Design

Bewusste Entscheidungen.

—
Schwarz, Weiß & Gold

Drei Farben, konsequent durchgezogen. Gold als einziger Akzent — sparsam eingesetzt, dadurch wirkungsvoll. Kein buntes Chaos, kein generisches Blau.

—
Custom Cursor

Ein kleiner goldener Punkt der dem Mauszeiger folgt — mit LERP-Interpolation für eine weiche, organische Bewegung. Ein Detail das man spürt bevor man es bemerkt.

—
Scroll-Animationen

Sektionen blenden sanft ein wenn sie in den Viewport kommen — via IntersectionObserver. Kein JavaScript-Framework, keine externe Animation Library.

—
Responsive Design

Ein Breakpoint bei 768px — Desktop und Mobil. Grid-Layouts die von mehrspaltigen zu einspaltigem Aufbau wechseln. Ohne CSS Framework, ohne Bootstrap.

04 — Reflexion

Was ich mitgenommen habe.

Die Portfolio-Seite war mein erster größerer Schritt in die professionelle Webentwicklung. Nicht das komplexeste Projekt — aber das Fundament auf dem alles andere aufbaut.

Was ich unterschätzt hatte: wie viele Entscheidungen hinter einer simplen Seite stecken. Welche Schrift? Welches Layout? Wie animiere ich das? Jede Frage eine Lerngelegenheit.

Das Ergebnis ist live, es funktioniert, und ich kann es jedem Arbeitgeber zeigen. Das ist mehr wert als jedes Tutorial das ich hätte nachbauen können.
